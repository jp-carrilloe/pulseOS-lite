## Purpose
Define how the centralized agent registry is implemented and maintained while preserving folder-level agent ownership.

## Scope
- Build a single registry source of truth for agent discovery and routing.
- Keep canonical agent documents in their domain folders and execution folders.
- Enforce relationship-change updates across all affected files.

## Implementation Steps
1. Create registry file in `501_Agents_and_Workflows` with one record per active agent.
2. Add or update agent cards in the registry README for operator visibility.
3. Link each registry entry to the canonical agent document path.
4. Enable audit checks to prevent registry drift and missing relationship metadata.

## Relationship Change Update Protocol (Mandatory)
- Every time a new dependency or relationship is added, update all of the following in the same change set:
- The agent document that introduced the new relationship.
- The directly related upstream and downstream agent documents.
- The central registry entry for each affected agent.
- Recommended file-read lists for each affected agent.
- Dependency and ownership sections in `README_Agents` and relevant folder readmes when routing changes.

## Required Agent Document Fields
- Dependencies
- Recommended File Reads
- Upstream Dependencies
- Downstream Dependents
- Change Trigger Notes

## Acceptance Criteria
- No relationship may exist only in one file.
- Registry and agent docs resolve to the same dependency graph.
- Docs audit fails if required relationship fields are missing in any agent document.
