## Purpose
This folder contains the canonical company brain.

Use this area for durable, reviewed company knowledge after source material has been ingested, seeded, or routed by agents.

The root of the repo stays focused on onboarding, source intake, and local tooling. The canonical company memory lives here.

---

## Folder Map
- `000_Agent_Shortcuts` — quick access symlinks to canonical domain agents.
- `101_System_Overview` — ARK, governance, document standards, and system architecture.
- `102_Corporate_Strategy_and_Foundation` — strategy spine, mission, offers, brand, and pricing.
- `103_Corporate_Operations` — operations, people, SOPs, vendors, and meeting records.
- `104_Finance_and_Financial_Planning` — budgets, financial models, and forecasts.
- `105_Technical_Infrastructure_and_Security` — systems, APIs, privacy, and security.
- `106_Legal_and_Compliance` — contracts, legal frameworks, compliance, and risk.
- `201_Market_Intelligence_and_ICP` — ICP, competitive intelligence, and market signals.
- `202_Go-to-Market_Strategy` — GTM foundation, channels, offers, and experiments.
- `203_Sales_Enablement_Hub` — pitch assets, playbooks, outreach, and enablement.
- `301_Client_Delivery_and_Onboarding` — implementation, onboarding, and delivery.
- `302_Analytics_and_Performance_Intelligence` — KPIs, reporting, and optimization.
- `401_Strategic_Partnerships` — partnerships, alliances, and channel relationships.
- `402_Fundraising` — fundraising strategy, data room, investor FAQ, and investor CRM.
- `501_Agents_and_Workflows` — agent registry, workflow governance, and conversation memory.
- `502_Execution_Engine` — local daemon, execution tooling, and integration scaffolding.
- `600_Projects` — project planning, PRDs, specifications, implementation notes, and project templates.

## Usage Rules
- Keep raw unreviewed source material in `../001_Source_Intake`.
- Keep durable company knowledge in this folder.
- Use `@ARK` for cross-domain routing.
- Use the domain owner agents before making specialist edits.
- Keep project implementation details in `600_Projects` unless they belong in a canonical domain folder.
