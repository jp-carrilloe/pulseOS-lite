## 2. Agent-Generated IP

As [CLIENT_NAME] deploys AI agents to produce deliverables, the following rules apply:

- **Outputs belong to the client** once full payment is received, per the MSA
- **The agent infrastructure, prompts, and workflows** that produced those outputs belong to [CLIENT_NAME] and are never transferred
- **Training data** used to fine-tune any model on client-specific data must be defined in the engagement SOW — default is client data remains client property
- **No deliverable produced by an agent** gives the client any right to the underlying AI systems, models, or automation logic used to produce it

> This distinction must be explicitly stated in every MSA and SOW. Never allow a client to claim ownership of the production infrastructure through ownership of the output.

---

## 3. Work-for-Hire vs. Licence-to-Use

**Work-for-Hire (Full Transfer):**
- Applies when the client commissions a fully custom asset (e.g., bespoke software, custom content, proprietary data model built exclusively for them)
- On full payment, [CLIENT_NAME] assigns all rights and has no residual claim
- Must be explicitly stated in the SOW

**Licence-to-Use:**
- Applies when [CLIENT_NAME] builds something reusable across multiple clients or retains the methodology
- Client receives a perpetual, non-exclusive licence to use the output
- [CLIENT_NAME] retains ownership and may use the underlying system for other clients
- Default model — use unless the SOW explicitly states otherwise

> Default to Licence-to-Use. Only use Work-for-Hire when a client specifically requests and pays for exclusivity.

## 5. Employee and Contractor IP Assignment

All employees and contractors must sign an IP assignment clause (included in employment/contractor agreements in `106.1_Contracts_and_Agreements`) confirming that:

- Any work created in the course of their engagement is owned by [CLIENT_NAME]
- Pre-existing personal IP remains theirs — but must be declared upfront if it will be used in client work
- Open-source contributions made during engagement require prior written approval

## 6. Open Source Policy

- Use of open-source libraries and tools is permitted provided the licence is compatible with commercial use
- **Copyleft licences (GPL, AGPL)** require legal review before use in client-facing products
- Maintain a registry of open-source dependencies and their licences
