## Purpose
This folder stores project-level work that should remain connected to the company brain but does not belong directly inside Strategy, Operations, GTM, Sales, Delivery, Finance, Legal, or Infrastructure.

Use it for:
- project planning
- PRDs
- implementation specifications
- project data models
- project runbooks
- project retrospectives
- reusable project templates

Do not put private client data, credentials, API keys, or personal information here.

---

## Folder Structure
- `600.1_Project_Index` — active and archived project inventory.
- `600.2_Project_Planning` — project briefs, goals, milestones, scope, and decision logs.
- `600.3_PRDs` — product requirement documents and feature briefs.
- `600.4_Specifications` — technical, operational, workflow, and integration specs.
- `600.5_Data_Model` — project schemas, entity definitions, field maps, and data contracts.
- `600.6_Operations` — runbooks, launch checklists, progress logs, and review notes.
- `600.7_Templates` — reusable project templates.

---

## Project Folder Convention
When a project becomes large enough to need its own folder, create it as:

`600_Projects/6XX_Project_Name/`

Recommended internal structure:
- `01_Planning`
- `02_PRDs`
- `03_Specifications`
- `04_Data_Model`
- `05_Operations`
- `06_Templates`

This structure is inspired by a larger project workspace pattern, but this repo includes only clean templates and no private project data.
