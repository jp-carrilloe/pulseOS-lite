# Lovable Website And Tracking Build Brief

## Purpose
Provide build instructions for creating the Migma GTM campaign website in Lovable, including landing page structure, qualification form logic, attribution capture, conversion tracking, CRM handoff, and retargeting readiness.

## Executive Summary
- Build one focused campaign site for the first segment: Klaviyo users.
- The page should sell one offer: a free Klaviyo Flow + Revenue Leak Audit.
- The form must qualify leads before they become sales-ready.
- Tracking must capture ad attribution, email attribution, LinkedIn attribution, website behavior, and qualification status.
- Browser tracking alone is not enough. The site needs server-side or webhook-based lead capture so qualified events can be sent to CRM, Meta, Google Ads, and Migma.

## Lovable Build Objective
Create a production-ready campaign website that:
- explains the Klaviyo audit offer clearly
- captures qualified inbound leads
- preserves attribution from PPC, Migma email, LinkedIn, and retargeting
- fires clean tracking events
- sends lead payloads to a CRM, database, or automation webhook
- supports future variants for Mailchimp and SendGrid

## Lead Scoring Logic
Implement this scoring in the form submit handler or webhook automation.

### Score Inputs
- Platform:
  - Klaviyo: `25`
  - Mailchimp: `15`
  - SendGrid: `15`
  - Other: `0`
- Company type:
  - Ecommerce: `25`
  - SaaS: `15`
  - Marketplace: `15`
  - Agency: `5`
  - Other: `0`
- Monthly email volume:
  - 250k+: `25`
  - 50k-250k: `20`
  - 10k-50k: `10`
  - Under 10k: `0`
- Role:
  - Founder / CEO: `20`
  - Growth / Marketing leader: `20`
  - Lifecycle / Email marketer: `15`
  - RevOps / Marketing Ops: `15`
  - Other: `0`
- Main problem:
  - Deliverability: `20`
  - Revenue leakage: `20`
  - Automation setup: `15`
  - Migration: `15`
  - Segmentation: `10`
  - Reporting: `10`

### Qualification Status
- `qualified_lead`:
  - score is `60+`
  - work email exists
  - website exists
- `sales_accepted_lead`:
  - score is `80+`
  - timeline is Now or This month
- `disqualified`:
  - score is below `40`
  - platform is Other
  - company type is Other
  - email is personal/non-business

## Tracking Requirements
### Google Tag Manager
Install GTM on every page.

Required environment variable or config value:
- `VITE_GTM_ID`

GTM script placement:
- Add GTM script in the page head.
- Add GTM noscript fallback immediately after opening body if Lovable exposes this.
- If Lovable does not expose raw body placement, document the limitation and install through the best supported global script area.

### Data Layer Events
Push these events to `window.dataLayer`.

#### Page View
Fire on route load:

```javascript
window.dataLayer = window.dataLayer || [];
window.dataLayer.push({
  event: 'page_view',
  page_path: window.location.pathname,
  page_location: window.location.href,
  page_title: document.title
});
```

#### Form Start
Fire once when the user first interacts with the form:

```javascript
window.dataLayer.push({
  event: 'lead_form_start',
  form_name: 'klaviyo_audit',
  page_path: window.location.pathname
});
```

#### Lead Submit
Fire after successful form submission:

```javascript
window.dataLayer.push({
  event: 'lead_submit',
  form_name: 'klaviyo_audit',
  lead_id: leadId,
  event_id: eventId,
  platform: form.platform,
  company_type: form.company_type,
  monthly_email_volume: form.monthly_email_volume,
  role: form.role,
  lead_score: leadScore,
  qualification_status: qualificationStatus
});
```

#### Qualified Lead
Fire only if the lead qualifies:

```javascript
window.dataLayer.push({
  event: 'qualified_lead',
  form_name: 'klaviyo_audit',
  lead_id: leadId,
  event_id: eventId,
  lead_score: leadScore,
  qualification_status: qualificationStatus,
  platform: form.platform,
  account_domain: accountDomain
});
```

### Meta Pixel
Install Meta Pixel through GTM or Lovable global scripts.

Required config:
- `VITE_META_PIXEL_ID`

Browser events:
- `PageView`
- `ViewContent`
- `Lead`
- `CompleteRegistration` or custom `QualifiedLead`

On raw lead submit:

```javascript
fbq('track', 'Lead', {
  content_name: 'Klaviyo Audit Form',
  platform: form.platform,
  lead_score: leadScore
}, {
  eventID: eventId
});
```

On qualified lead:

```javascript
fbq('trackCustom', 'QualifiedLead', {
  content_name: 'Klaviyo Audit Form',
  platform: form.platform,
  lead_score: leadScore,
  account_domain: accountDomain
}, {
  eventID: eventId
});
```

### Webhook Destinations
Send the payload to:
- CRM:
  - HubSpot, Attio, or Airtable
- Migma:
  - update contact state and suppress from active sequence when appropriate
- Meta CAPI:
  - send `Lead` and `QualifiedLead`
- Google Ads:
  - store data for offline conversion import
- Slack or email alert:
  - only for `sales_accepted_lead`

## Post-Submit Experience
### Qualified Lead
Show:
- `Thanks. Your audit request has been received.`
- `If your team is a fit, we will follow up with next steps.`

Optional:
- embed calendar only for `sales_accepted_lead`
- avoid showing a calendar to low-fit leads

### Unqualified Lead
Show:
- `Thanks. We received your request.`
- link to a useful resource
- do not trigger high-priority sales alerts

## Retargeting Audiences
Build audiences for:
- all visitors
- Klaviyo audit page visitors
- form starters
- form submitters
- qualified leads
- high-intent visitors
- target account visitors

Exclude:
- customers
- booked calls
- disqualified leads
- opted-out contacts
