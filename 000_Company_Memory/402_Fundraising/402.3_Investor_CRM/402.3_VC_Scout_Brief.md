## What It Does
VC Scout is an AI-powered investor discovery agent that automates finding, verifying, and filtering high-conviction investors for [CLIENT_NAME]. It orchestrates external APIs in a pipeline to go from a search query to a curated, enriched, de-duplicated investor shortlist.

**Script:** `scripts/vc_scout.py`

---

## The Three-Tool Pipeline

| Step | Tool | Purpose |
| :--- | :--- | :--- |
| 1 | **Exa Search** | Neural semantic discovery of VCs via portfolio similarity, publications, and social posts |
| 2 | **Perplexity** | Factual deep-dive to verify check sizes, fund status, recent activity, and conflict-of-interest screening |
| 3 | **Firecrawl** | Deep website crawling of investor pages, portfolio sites, and profile enrichment |

Each step feeds into the next. Firecrawl is optional (`--skip-firecrawl`) to save costs when speed matters more than depth.

## Conflict Classification

The script automatically classifies every investor's portfolio overlap:

| Class | Meaning | Action |
| :--- | :--- | :--- |
| 🔴 `hard` | Active, direct competitor in portfolio | Flag and skip — fiduciary conflict |
| 🔥 `warm` | Exited or adjacent company in portfolio | Prioritize — proven thesis on this space |
| 🟡 `flag` | Legacy or tangentially related portfolio company | Keep with note — may understand the pain |
| ✅ `clear` | No overlap found | Include as standard lead |

**Hard conflicts (define for [CLIENT_NAME]):** [INSERT_DIRECT_COMPETITORS]
**Warm signals (define for [CLIENT_NAME]):** [INSERT_ADJACENT_COMPANIES]
**Flag signals (define for [CLIENT_NAME]):** [INSERT_LEGACY_PLAYERS]

## CLI Usage

```bash
# Run all built-in seed queries (recommended starting point)
python3 402_Fundraising/402.3_Investor_CRM/scripts/vc_scout.py --run-all

# Run a custom free-form discovery query
python3 402_Fundraising/402.3_Investor_CRM/scripts/vc_scout.py \
    --query "[INSERT_CUSTOM_QUERY]"

# Deep-dive a specific investor
python3 402_Fundraising/402.3_Investor_CRM/scripts/vc_scout.py \
    --enrich "[INSERT_INVESTOR_NAME, FUND_NAME]"

# Crawl a specific firm website
python3 402_Fundraising/402.3_Investor_CRM/scripts/vc_scout.py \
    --crawl-url "[INSERT_FIRM_URL]"

# Faster, cheaper run — skips Firecrawl enrichment
python3 402_Fundraising/402.3_Investor_CRM/scripts/vc_scout.py \
    --run-all --skip-firecrawl
```

### Key Flags

| Flag | Default | Description |
| :--- | :--- | :--- |
| `--run-all` | — | Run all built-in seed queries sequentially |
| `--query` | — | Custom free-form Exa discovery query |
| `--enrich` | — | `"Partner Name, Fund"` — deep-dive a single investor |
| `--crawl-url` | — | Crawl a specific URL via Firecrawl |
| `--exa-results` | `10` | Number of Exa results per query |
| `--perplexity-model` | `sonar` | Perplexity model to use |
| `--skip-firecrawl` | `false` | Skip Firecrawl step (faster, cheaper) |
| `--skip-conflicts` | `false` | Skip conflict classification |

## Outputs

All outputs are written to `402_Fundraising/402.3_Investor_CRM/output/`:

| File | Format | Contents |
| :--- | :--- | :--- |
| `vc_scout_results.json` | JSON | Full structured investor data, merged across runs |
| `vc_scout_results.md` | Markdown | Human-readable report with tables and profiles |
| `vc_scout_api_usage.json` | JSON | Per-call API usage log with token counts and cost estimates |

Results are **merged across runs** — re-running adds new investors without removing previously found ones.
